# ASIF: Adaptive Soft Identity Framework
**Adaptive Soft Identity Framework for Cybersecurity Risk Detection in Digital Financial Systems**

---

## ที่มาและแนวคิดของโครงการ (Project Overview)
ในปัจจุบัน ระบบยืนยันตัวตนแบบ **Hard Identity** (เช่น OTP, รหัสผ่าน หรือ Biometrics ลายนิ้วมือ/ใบหน้า) อาจไม่เพียงพอต่อการป้องกันภัยคุกคามสมัยใหม่ เช่น บัญชีม้า หรือการถูกทำ Social Engineering เนื่องจากหากคนร้ายครอบครองข้อมูลเหล่านั้นได้ ระบบจะไม่สามารถแยกแยะได้เลยว่าผู้ใช้งานคือเจ้าของตัวจริงหรือไม่ 

**ASIF** จึงนำเสนอแนวคิด **Soft Identity (อัตลักษณ์เชิงพฤติกรรม)** ซึ่งเป็นการใช้ AI/ML เรียนรู้พฤติกรรมเฉพาะตัวของผู้ใช้งานในหลายมิติ เช่น:
* **Typing Patterns:** รูปแบบการพิมพ์ 
* **Touch Dynamics:** ลักษณะการสัมผัสหน้าจอ 
* **Usage Context:** ช่วงเวลา สถานที่ และอุปกรณ์ที่ใช้เป็นประจำ 

เป้าหมายคือการสร้าง **Baseline Soft Identity** เพื่อตรวจจับความเบี่ยงเบนของพฤติกรรม (Deviation) และประเมินออกมาเป็น **Risk Score** เพื่อแจ้งเตือนหรือเพิ่มขั้นตอนการยืนยันตัวตนก่อนเกิดความเสียหาย

## ขั้นตอนการทดลองใช้งาน Demo (Experimental Steps)
*หมายเหตุ: ส่วนนี้เป็นชุดโปรแกรมสำหรับการทดลอง (Experiment) เพื่อจำลองการเรียนรู้พฤติกรรมเบื้องต้นเท่านั้น อาจจะเกิดข้อผิดพลาดบางประการได้ หรือความเเม่นยำที่ยังไม่มากพอ*

โครงสร้างไฟล์ใน Demo:
- `/core`: เก็บไฟล์ Source Code (.py)
- `/demo`: เก็บไฟล์ที่ Execute ได้ทันที (.exe)

### 1. การสร้างข้อมูลสำหรับ Train (Data Collection)
เข้าใช้งานไฟล์ **`SelfTrainDataApplication`** เพื่อสร้างข้อมูลพฤติกรรมเริ่มต้น
* พิมพ์ข้อความในกรอบที่กำหนดเพื่อให้ระบบเก็บตัวอย่างพฤติกรรม
* กด **Save** ระบบจะบันทึกข้อมูลเป็นไฟล์ **JSON**

### 2. การเทรนโมเดล (Model Training)
เข้าใช้งานไฟล์ **`MainEngineForStd-Train`** (ต้องอยู่ใน Directory เดียวกับไฟล์ JSON จากข้อ 1)
* ระบบจะนำข้อมูลพฤติกรรมมาเรียนรู้และสร้างโมเดล
* เมื่อเสร็จสิ้นจะได้ไฟล์โมเดลนามสกุล **`.pkl`** ออกมา

### 3. การสร้างข้อมูลสำหรับตรวจสอบ (Testing Data)
กลับไปที่ไฟล์ **`SelfTrainDataApplication`** อีกครั้งเพื่อสร้างข้อมูลชุดใหม่
* **กรณีทดสอบว่าเป็นตัวเอง:** ให้พิมพ์ด้วยจังหวะปกติของตนเองแล้วกด Save
* **กรณีทดสอบว่าเป็นคนอื่น:** ให้ลองให้เพื่อนหรือผู้อื่นมาพิมพ์แทนแล้วกด Save

### 4. การตรวจสอบผลลัพธ์ (Verification)
เข้าใช้งานไฟล์ **`Verify`** เพื่อดูผลการประเมิน
* ระบบจะเปรียบเทียบข้อมูลใหม่กับไฟล์โมเดล (.pkl)
* แสดงผลลัพธ์ว่าพฤติกรรมที่ตรวจพบนั้น ตรงกับเจ้าของอัตลักษณ์ (Soft Identity) หรือไม่
